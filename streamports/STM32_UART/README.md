# STM32 HAL UART Stream Port for Percepio TraceRecorder

A custom stream port that sends trace data over an STM32 HAL UART peripheral,
enabling bidirectional streaming with Tracealyzer via a serial connection.

Compatible with **TraceRecorder v4.8+** (tested against the v4.10 / v4.11 API).

## Directory structure

```
streamport_uart/
├── config/
│   └── trcStreamPortConfig.h   ← Tunable settings (timeouts, buffer size)
├── include/
│   └── trcStreamPort.h         ← Stream port types & macros
├── trcStreamPort.c             ← UART read/write implementation
└── README.md
```

## Quick-start

### 1. Copy into your project

Copy the three source/header files into your TraceRecorder tree, for example:

```
TraceRecorder/streamports/STM32_HAL_UART/config/trcStreamPortConfig.h
TraceRecorder/streamports/STM32_HAL_UART/include/trcStreamPort.h
TraceRecorder/streamports/STM32_HAL_UART/trcStreamPort.c
```

Add `trcStreamPort.c` to your build, and add both `config/` and `include/`
to your compiler include paths.

### 2. Define `TRACE_UART`

In your `trcConfig.h` (or a header included before it), add:

```c
#define TRACE_UART huart1          /* or huart2, huart3, … */
```

This must match the `UART_HandleTypeDef` variable generated by STM32CubeMX.

### 3. Include your processor header

In `trcConfig.h`, replace the `#error` reminder line with your HAL include:

```c
#include "stm32f4xx_hal.h"         /* adjust to your STM32 family */
```

This makes `UART_HandleTypeDef`, `HAL_UART_Transmit`, etc. visible to the
stream port code.

### 4. Initialise the UART before tracing

The UART must be fully initialised before `xTraceEnable()` is called.
With CubeMX-generated code this is normally the case since `MX_USARTx_UART_Init()`
runs early in `main()`:

```c
int main(void)
{
    HAL_Init();
    SystemClock_Config();

    MX_USART1_UART_Init();       /* UART ready */

    xTraceEnable(TRC_START);     /* or TRC_START_FROM_HOST / TRC_START_AWAIT_HOST */

    /* ... create tasks, start scheduler ... */
}
```

### 5. Connect Tracealyzer

In Tracealyzer, go to **PSF Streaming Settings** and select **Serial Port**.
Set the COM port and baud rate to match your UART configuration.

## Configuration options

All tunables live in `config/trcStreamPortConfig.h` and can be overridden
from `trcConfig.h`.

| Define | Default | Description |
|--------|---------|-------------|
| `TRACE_UART` | *(must be set)* | The `UART_HandleTypeDef` instance name |
| `TRC_CFG_STREAM_PORT_USE_INTERNAL_BUFFER` | `1` | Use the recorder's internal event buffer (recommended) |
| `TRC_CFG_STREAM_PORT_INTERNAL_BUFFER_SIZE` | `5120` | Internal buffer size in bytes |
| `TRC_CFG_STREAM_PORT_UART_TRANSMIT_TIMEOUT_MS` | `5` | `HAL_UART_Transmit` timeout per call |
| `TRC_CFG_STREAM_PORT_UART_RECEIVE_TIMEOUT_MS` | `1` | `HAL_UART_Receive` timeout per byte |

## Baud rate considerations

Typical RTOS trace data rates range from 20–200 KB/s depending on system
activity. At 115200 baud (~14.4 KB/s) you will almost certainly overflow
the buffer. Recommended minimum baud rates:

- **230400** — tight but workable for light workloads
- **460800** — comfortable for most systems
- **921600** — recommended for production use

Configure the baud rate in CubeMX / your UART init code; the stream port
itself is baud-agnostic.

## How it works

| Function | Role |
|----------|------|
| `xTraceStreamPortInitialize()` | Called once during `xTraceEnable()`. No-op since the HAL UART is already running. |
| `xTraceStreamPortWriteData()` | Sends a buffer of trace data via `HAL_UART_Transmit` (blocking, short timeout). |
| `xTraceStreamPortReadData()` | Polls for incoming host commands via `HAL_UART_Receive` (1 byte at a time, short timeout). |

Both read and write are called from the **TzCtrl** task context, so they may
use RTOS primitives. The internal event buffer decouples the high-frequency
event generation from the lower-frequency UART flushes.

## Extending to DMA

For higher throughput or lower CPU overhead you can replace the blocking
`HAL_UART_Transmit` / `HAL_UART_Receive` calls in `trcStreamPort.c` with
their DMA variants (`HAL_UART_Transmit_DMA`, `HAL_UART_Receive_DMA`) and
a completion callback. The header and config files remain unchanged.
